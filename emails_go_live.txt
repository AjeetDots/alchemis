Where ALCHEMIS_ENV (from .env) is used – no file changes needed for go-live
--------------------------------------------------------------------------------
• **index.php** – Reads ALCHEMIS_ENV from .env (if not set by server), then sets error display (dev vs live) and passes env to the app. Default if unset: 'aws'.
• **db_test.php** – Reads ALCHEMIS_ENV from .env; chooses DB host (e.g. "db" for local, or RDS for aws).
• **ApplicationHelper.php** – Uses $_SERVER['ALCHEMIS_ENV'] to load the right block from app_options.xml (development vs aws) and set DB connection.
• **app/base/Registry.php** – Uses $_SERVER['ALCHEMIS_ENV'] for config freeze file and environment.
• **data/app_options.xml** – Contains <development> and <aws> blocks; the value of ALCHEMIS_ENV selects which block is used. No change needed; live uses <aws> when ALCHEMIS_ENV=aws.
• **docker-compose.yml** – Can set ALCHEMIS_ENV=development for local app container; for live, set ALCHEMIS_ENV=aws if you run Docker on the server.

For go-live: set ALCHEMIS_ENV=aws on the server (via .env or server/env). No need to edit index.php, ApplicationHelper, or app_options.xml.

Local Docker: if legacy_db has no tables (blank login / "No tables found")
--------------------------------------------------------------------------------
The DB is on localhost; the database legacy_db is just empty. Import data so the app has tables and users.

**Option A – Import from live backup (recommended; you get real data and users):**
1. Ensure Docker is running and container legacy_mysql is up.
2. From project root:  php import_live_to_local.php
   (Uses the latest backup_live_*.sql in the project, e.g. backup_live_20260211_124400.sql.)
3. Or with a specific file:  php import_live_to_local.php path/to/dump.sql
4. Then open http://localhost:8080/index.php?cmd=Login again.

**Option B – Fresh schema only (empty tables, no users):**
Import data/schema.sql into legacy_db (e.g. via phpMyAdmin Import, or docker exec -i legacy_mysql mysql -u legacy_user -psecret legacy_db < data/schema.sql). You will still need to create a user or import data for login to work.
